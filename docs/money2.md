# Money Plan v2 (на базе `money.md`)

Документ фиксирует конкретный план монетизации из `money.md` (этап 1: `Freemium + Premium` и минимальная нативная реклама), и раскладывает функциональность по тарифам:

- что уже реализовано в текущем продукте;
- что нужно реализовать для запуска монетизации.

---

## 1) Тарифная сетка

### Пользователи
- `Free User`
- `Premium User` (299 ₽/мес или 2 490 ₽/год, trial 7 дней)

### Тренеры
- `Free Coach` (до 5 активных клиентов)
- `Coach Pro` (990 ₽/мес или 9 900 ₽/год, trial 7 дней)

---

## 2) Функциональность по тарифам

## `Free User`
**Доступно по тарифу**
- Базовые тренировки (создание, выполнение, завершение).
- Шаблоны тренировок.
- Базовый прогресс и замеры.
- Социальная лента.
- Геймификация в базовом виде (по feature flags).
- Нативная реклама в контентных экранах.

**Уже реализовано**
- Тренировки, шаблоны, прогресс, замеры, лента, геймификация.
- Базовая модель подписки в профиле пользователя (`paidSubscriber`, `subscriptionExpiresAt`) присутствует.

**Нужно реализовать**
- Ограничение аналитики для free (например, история только 14 дней).
- Ограничение по персональным целям (например, 1 активная цель).
- Рекламная подсистема (плейсменты, частотный лимит, таргетинг free-only).
- Правила показа рекламы: только safe-экраны, без active workout.

---

## `Premium User`
**Доступно по тарифу**
- Все функции `Free User`.
- Полная история аналитики (без ограничений по периоду).
- Расширенные цели и отчеты.
- Расширенная геймификация.
- Реклама отключена.

**Уже реализовано**
- Основа для статуса подписки в профиле и на backend (поля подписки есть).
- Большая часть функционала, который можно “раскрывать” по paywall, уже есть в продукте.

**Нужно реализовать**
- Централизованный entitlement-check (`isPremium`) на клиенте и сервере.
- Paywall и триггеры в ключевых местах:
  - попытка открыть расширенную аналитику;
  - история > 14 дней;
  - запуск premium-целей.
- Тарифная логика trial/renew/cancel.
- Полное скрытие рекламы для premium.

---

## `Free Coach`
**Доступно по тарифу**
- Базовый кабинет тренера.
- До 5 активных клиентов.
- Базовые отчеты по клиентам.
- Реклама только в нерабочих экранах.

**Уже реализовано**
- Кабинет тренера, подопечные, профили клиентов, прогресс клиентов.
- Групповые шаблоны и групповые тренировки.

**Нужно реализовать**
- Лимит в 5 активных клиентов на уровне backend + UI сообщений.
- Разделение “базовые/расширенные отчеты”.
- Ограничения на экспорт/доп. инструменты для free coach.
- Логика рекламы для coach-only нерабочих экранов.

---

## `Coach Pro`
**Доступно по тарифу**
- Все функции `Free Coach`.
- Безлимит клиентов.
- Расширенные отчеты и экспорт.
- Расширенные инструменты шаблонов/программ.
- Приоритет в каталоге тренеров.
- Реклама отключена.

**Уже реализовано**
- Большая часть функционала кабинета тренера, которую можно сделать Pro-only.

**Нужно реализовать**
- Entitlement `isCoachPro` в backend и mobile.
- Снятие лимита по клиентам.
- Расширенные отчеты/экспорт (как отдельные endpoint + UI).
- Механика приоритета в каталоге тренеров (ранжирование).
- Полное отключение рекламы.

---

## 3) Реклама (этап 1)

**Цель:** минимальная нативная реклама для free, без ущерба core UX.

### Плейсменты
- Лента (`/feed`): 1 рекламный блок на 8-10 карточек.
- Контентные списки (нерабочие экраны тренера).

### Где запрещено
- Активная тренировка (`/workout/:id`).
- Логирование подходов.
- Финиш тренировки и экран итогов.
- Редактирование клиентского плана.

### Ограничения
- Frequency cap: до 3 показов/сутки на пользователя.
- Показ только для `Free User` и `Free Coach`.

### Статус
- **Реализовано:** нет.
- **Нужно:** ad provider, ad slots, caps, сегментация по тарифам, мониторинг eCPM/retention.

---

## 4) Paywall и биллинг

### Что нужно сделать в первую очередь
1. Единая сущность тарифа:
   - `user_plan`: free/premium
   - `coach_plan`: free/pro
   - trial state, expires_at, renew state
2. Middleware/guard на backend для premium/pro функций.
3. UI paywall:
   - user paywall;
   - coach paywall.
4. События аналитики:
   - `paywall_view`, `trial_start`, `purchase_success`, `purchase_fail`, `churn`.

### Текущий статус
- **Частично реализовано:** есть признаки подписки в профиле.
- **Не реализовано:** полноценная монетизационная матрица прав и платежный контур.

---

## 5) Конкретный план внедрения (4 спринта)

## Спринт 1
- Ввести план/entitlements в backend.
- Добавить guards для premium/pro.
- Подготовить UI paywall (user + coach, без оплаты).

## Спринт 2
- Подключить оплату и trial.
- Включить ограничения free:
  - user: история аналитики 14 дней;
  - coach: максимум 5 активных клиентов.
- Запустить premium/pro разблокировки.

## Спринт 3
- Включить нативную рекламу для free сегментов.
- Реализовать frequency cap и safe-zones.
- Включить отключение рекламы для paid.

## Спринт 4
- A/B тесты paywall и цен.
- Улучшение конверсии trial -> paid.
- Подготовка к этапу 2 (`разовые покупки программ`).

---

## 6) KPI запуска

- Trial conversion:
  - User >= 6%
  - Coach >= 8%
- Trial -> Paid:
  - User >= 35%
  - Coach >= 45%
- Paid share:
  - User >= 2.5%
  - Coach >= 8%
- Ad ARPDAU free: 0.8-1.2 ₽
- D30 retention drop после рекламы: не более 2 п.п.
