# Функциональное описание приложения FitFlow

Ниже — деревянный список экранов мобильного клиента (Flutter, `go_router`), публичных лендингов, REST API (`/api/v1`) и веб-админки (`/admin`). Базовый URL API и хост приложения зависят от окружения.

---

## Мобильное приложение (Flutter)

### Загрузка и авторизация

- **Загрузка** (`/loading`) — проверка сессии, редирект на `/home` или `/login`.

  **Функциональность**

  - [Инициализация] — определение, авторизован ли пользователь, переход в основное приложение или на вход.

- **Вход** (`/login`) — авторизация по учётной записи.

  **Функциональность**

  - [Вход] — аутентификация, переход в приложение (`/home`).
  - [Регистрация] — переход на `/register`.

- **Регистрация** (`/register`) — создание новой учётной записи.

  **Функциональность**

  - [Регистрация] — отправка данных регистрации, вход в приложение после успеха.
  - [Вход] — возврат на `/login`.

### Основная оболочка (нижняя навигация и общий layout)

Экраны ниже открываются внутри `MainShellScreen` (кроме отдельно перечисленных полноэкранных маршрутов).

- **Главная — список тренировок** (`/home`) — обзор тренировок пользователя.

  **Функциональность**

  - [Список тренировок] — просмотр и управление запланированными/текущими тренировками.
  - [Переход к тренировке] — открытие активной тренировки (`/workout/:id`).

- **Календарь** (`/calendar`) — календарь событий и тренировок.

  **Функциональность**

  - [Календарь] — навигация по датам, просмотр занятий.

- **Упражнения** (`/exercises`) — каталог упражнений.

  **Функциональность**

  - [Каталог] — поиск и просмотр упражнений.

- **Шаблоны тренировок** (`/templates`) — пользовательские шаблоны программ.

  **Функциональность**

  - [Список шаблонов] — создание, выбор и редактирование шаблонов.
  - [Редактор шаблона] (`/templates/:id/edit`) — название, упражнения, подходы, старт тренировки из шаблона.
  - [Выбор упражнения] (`/templates/:id/pick-exercise`) — добавление упражнения в шаблон.

- **Текущая тренировка** (`/current-workout`) — краткий вид активной сессии.

  **Функциональность**

  - [Быстрый доступ] — переход к выполняемой тренировке.

- **Таймеры** (`/timers`) — таймеры для тренировки.

  **Функциональность**

  - [Таймеры] — отсчёт интервалов, паузы между подходами.

- **Профиль** (`/profile`) — профиль пользователя, настройки аккаунта.

  **Функциональность**

  - [Профиль] — данные пользователя, аватар, метрики (через API).
  - [Выход] — при необходимости — смена аккаунта через настройки/опции.

- **Прогресс** (`/progress`) — меню разделов прогресса.

  **Функциональность**

  - [Навигация] — переход к замерам, статистике тренировок, упражнениям, мышцам.

- **Замеры / метрики** (`/progress/measurements`) — вес, объёмы, связанные показатели.

  **Функциональность**

  - [Замеры] — ввод и история показателей тела.

- **Прогресс: тренировки** (`/progress/workouts`) — агрегированная статистика по тренировкам.

  **Функциональность**

  - [Статистика] — обзор объёма и активности по тренировкам.

- **Прогресс: упражнения** (`/progress/exercises`) — прогресс по отдельным упражнениям.

  **Функциональность**

  - [Упражнения] — история объёмов и показателей по упражнению.

- **Прогресс: мышцы** (`/progress/muscles`) — распределение нагрузки по группам мышц.

  **Функциональность**

  - [Мышцы] — визуализация/сводка по мышечным группам.

- **Лента** (`/feed`) — социальная лента постов.

  **Функциональность**

  - [Лента] — просмотр постов, лайки, комментарии (через API соц. модуля).

- **Системные сообщения** (`/system-messages`) — объявления от платформы.

  **Функциональность**

  - [Сообщения] — список активных системных уведомлений.

- **Мои групповые тренировки** (`/group-trainings`) — записи пользователя на групповые занятия.

  **Функциональность**

  - [Мои записи] — список предстоящих и прошедших групповых тренировок.
  - [Детали] (`/group-trainings/:trainingId`) — информация о занятии, отмена записи при необходимости.

- **Доступные групповые тренировки** (`/group-trainings/available`) — каталог занятий для записи.

  **Функциональность**

  - [Поиск и запись] — выбор занятия, регистрация.

- **Справка** (`/help`) — раздел помощи.

  **Функциональность**

  - [Темы] — список тем.
  - [Тема] (`/help/:topicId`) — текст справки по выбранной теме.

- **Активная тренировка** (`/workout/:id`) — экран выполнения тренировки.

  **Функциональность**

  - [Выполнение] — подходы, веса, завершение тренировки.
  - [Только просмотр] — режим `readOnly=1` в query (история без редактирования).

- **Статистика тренировки** (`/workout/:id/stats`) — итоги завершённой сессии.

  **Функциональность**

  - [Итоги] — сводка по тренировке.

### Роль тренера (раздел `/trainer/...`)

- **Профиль тренера** (`/trainer`, `/trainer/profile`) — публичная и редактируемая информация тренера.

  **Функциональность**

  - [Профиль] — описание, контакты, фото портфолио.
  - [Редактирование] (`/trainer/profile/edit`) — изменение данных профиля тренера.

- **Клиенты** (`/trainer/trainees`) — список подопечных.

  **Функциональность**

  - [Список] — поиск и управление клиентами.
  - [Карточка клиента] (`/trainer/trainees/:clientId`) — профиль подопечного.
  - [Прогресс клиента] (`/trainer/trainees/:clientId/progress`) — объёмы и история по упражнениям.

- **Календарь тренера** (`/trainer/calendar`) — в роутере перенаправляется на общий `/calendar`.

- **Шаблоны групповых тренировок** (`/trainer/group-training-templates`) — шаблоны для проведения групповых занятий.

  **Функциональность**

  - [Список шаблонов] — создание, редактирование, удаление (мягкое на бэкенде).
  - [Новый шаблон] (`/trainer/group-training-templates/new`) — мастер создания.
  - [Редактор шаблона] (`/trainer/group-training-templates/:templateId`) — название, тип, описание, загрузка фото обложки.

- **Групповые тренировки тренера** (`/trainer/group-trainings`) — конкретные проводимые занятия.

  **Функциональность**

  - [Список занятий] — расписание и статусы.
  - [Новое занятие] (`/trainer/group-trainings/new`) — создание по шаблону/параметрам.
  - [Детали] (`/trainer/group-trainings/:trainingId`) — участники, информация.
  - [Редактирование] (`/trainer/group-trainings/:trainingId/edit`) — изменение времени, зала, лимитов.

### Экраны вне нижней оболочки (корневые дочерние маршруты `/`)

- **Залы** (`/gym`) — поиск и список фитнес-центров.

  **Функциональность**

  - [Поиск залов] — загрузка, фильтры (через API залов).
  - [Карточка зала] (`/gym/:gymId`) — детали, загрузка зала, тренеры/групповые тренировки в зале.

- **Мои тренеры** (`/my-trainers`) — связь с тренерами.

  **Функциональность**

  - [Список тренеров] — добавление и удаление своих тренеров.

- **Настройки / опции** (`/options`) — опции приложения.

  **Функциональность**

  - [Настройки] — язык, уведомления, прочие предпочтения.

---

### Публичные страницы (без обязательной авторизации)

- **Публичный профиль тренера** (`/t/:userId`) — лендинг тренера для шаринга ссылки.

  **Функциональность**

  - [Просмотр] — информация о тренере, предстоящие групповые тренировки (публичный API).

- **Публичная групповая тренировка** (`/g/:trainingId`) — лендинг занятия для записи или просмотра.

  **Функциональность**

  - [Инфо о занятии] — время, зал, тренер; запись возможна после входа.

## REST API (`/api/v1`)

Базовый префикс: **`/api/v1`**. Ниже — сгруппированные возможности (не дублируется каждый HTTP-метод в дереве).

- **Служебное** — `GET /health`, `GET /health/ready`, `GET /health/live`, `GET /version`, `GET /openapi.yaml`, `GET /ping`, локали `GET /locales`, `GET /locales/:lang`.

- **Аутентификация (без JWT)** — `POST /auth/register`, `POST /auth/login`, `POST /auth/refresh` (лимит запросов по IP).

- **Публично** — поиск залов `GET /gyms`, загрузка зала `GET /gyms/:gym_id/load`, история загрузки; тренеры в зале и групповые тренировки зала; публичный профиль тренера `GET /trainers/:user_id`; публичная групповая тренировка `GET /group-trainings/:training_id`; предстоящие групповые у тренера `GET /trainers/:user_id/group-trainings/upcoming`; блог `GET /blog-posts`, `GET /blog-posts/:post_id`, теги `GET /tags`.

- **С JWT (`Authorization: Bearer`)** — текущий пользователь `GET /me`, `PATCH /me/preferences`; профиль и метрики `/users/me/...`; аватар; гео-прокси `/geo/...` при наличии клиента; «мои залы», чек-ины; тренировки, упражнения, программы, шаблоны тренировок, логи подходов; прогресс (вес, % жира, health metrics); соцсеть (подписки, друзья, лента, посты, лайки, комментарии); блог автора (CRUD постов, фото, теги); роль тренера (профиль, клиенты, программы, комментарии к клиентам); уведомления; системные сообщения; загрузка фото в объектное хранилище `POST /me/photos/upload`; групповые тренировки (типы, шаблоны тренера, создание/редактирование занятий, запись пользователя на занятия).

- **Админ API (JWT + роль admin)** — `GET /admin/ping`, `POST /admin/gyms` (создание зала).

Статические загрузки пользователей: **`GET /uploads/...`** при настроенном `UploadsPath`.

## Веб-админка (`/admin`)

HTML-интерфейс на том же бэкенде (часто за nginx по отдельному хосту).

- **Вход** (`GET/POST /admin/login`, `GET /admin`) — авторизация администратора.

  **Функциональность**

  - [Логин] — сессия админа, редирект в кабинет.

- **Защищённая зона** — `POST /admin/logout`, `GET /admin/dashboard`.

  **Функциональность**

  - [Дашборд] — обзор после входа.
  - [Пользователи] — список, создание/редактирование (`/admin/entities/users/...`).
  - [Залы] — CRUD залов.
  - [Упражнения] — CRUD упражнений.
  - [Программы] — CRUD программ тренировок.
  - [Теги] — управление тегами.
  - [Посты блога] — CRUD постов.
  - [Системные сообщения] — рассылка/тексты для приложения.
  - [Buckets] — настройка бакетов объектного хранилища.
  - [Photos] — просмотр и управление записями о загруженных фото.
